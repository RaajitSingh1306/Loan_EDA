# Credit Default Predictor

A full end-to-end ML project that predicts whether a loan applicant will default, with a REST API and interactive UI.

---

## What this project does

Given a set of applicant details (age, income, loan amount, credit score, etc.), the model outputs:
- A **default probability** (0–1)
- A **label** — Approved or Default
- **SHAP values** — explaining *why* the model made that decision

---


## The ML Pipeline (main.ipynb)

### Phase 1 — EDA & Cleaning
- Dataset: ~45,000 loan applications
- Dropped impossible ages (>100) and employment experience outliers
- Capped income at 99th percentile (preserves data, handles extreme skew)
- Explored distributions, default rates by category, correlation heatmap

### Phase 2 — Encoding
- **Binary** (gender, prior default): LabelEncoder → 0/1
- **Ordinal** (education: High School < Associate < Bachelor < Master < Doctorate): OrdinalEncoder
- **Nominal** (home ownership, loan intent): one-hot with `drop_first=True` to avoid multicollinearity

### Phase 3 — Modelling
- Baseline: Decision Tree (shallow, depth=5) — interpretable, used to understand feature splits
- Final model: **Random Forest** (100 trees, `class_weight='balanced'`)
  - `class_weight='balanced'` is critical here — the dataset is imbalanced (~22% defaults), so without it the model ignores the minority class

### Phase 4 — Walk-Forward CV
Uses `TimeSeriesSplit` (5 folds) instead of random StratifiedKFold:
- **Fold 1**: Train on rows 0–9k, test on 9k–18k
- **Fold 2**: Train on rows 0–18k, test on 18k–27k
- ...and so on (expanding window)

Why? This simulates real deployment: you always train on past data and predict on future applicants. Random CV leaks future data into training, which inflates scores.

### Phase 5 — SHAP
- `shap.TreeExplainer` computes feature contributions for the RF model
- Summary plot shows global feature importance
- Per-prediction SHAP values exposed via API and visualised in the UI

## API (app.py)

Built with **FastAPI** + **Pydantic**.

```
POST /predict
```
Accepts 18 input features as JSON, returns:
```json
{
  "prediction": 1,
  "label": "Default",
  "default_probability": 0.73,
  "shap_values": { "loan_percent_income": 0.12, ... },
  "top_drivers": [
    { "feature": "loan_percent_income", "shap": 0.12 },
    ...
  ]
}
```

Run with:
```bash
uvicorn app:app --reload
```

---

## UI (streamlit_app.py)

Built with **Streamlit**.

- Form with all 18 input fields
- Calls the FastAPI `/predict` endpoint
- Shows risk label + probability
- Bar chart of top 5 SHAP drivers (red = increases default risk, green = reduces it)
- Expandable table of all SHAP values

Run with (in a second terminal):
```bash
streamlit run streamlit_app.py
```

## How to run the full stack

```bash
# Terminal 1 — API
uvicorn app:app --reload

# Terminal 2 — UI
streamlit run streamlit_app.py
```

Then open `http://localhost:8501` in your browser.

## Key concepts used

| Concept | Where | Why |
|---|---|---|
| Outlier handling | EDA | Garbage data corrupts model |
| Ordinal vs one-hot encoding | Encoding | Education has order; loan intent doesn't |
| class_weight='balanced' | RF model | Imbalanced classes → biased model without it |
| Walk-forward CV | Validation | Prevents future data leaking into training |
| SHAP | Explainability | Makes black-box RF interpretable per prediction |
| FastAPI + Pydantic | Deployment | Type-safe, auto-documented REST API |
| Streamlit | UI | Rapid ML frontend without frontend dev |

## Results

- **ROC-AUC** (walk-forward CV): ~0.93–0.95 across folds
- **Top predictive features**: `loan_percent_income`, `credit_score`, `previous_loan_defaults_on_file`, `loan_int_rate`, `loan_amnt`
